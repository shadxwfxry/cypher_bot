# Техническая документация Cypher Bot

Cypher Bot представляет собой клиентское программное обеспечение для интеграции с внешним маркетплейсом. Бот разработан на базе асинхронного фреймворка Aiogram 3 и предоставляет пользователям функционал управления личным кабинетом, пополнения баланса криптовалютой, просмотра истории покупок и транзакций, экспорта отчетности в фоновом режиме, а также мгновенного получения кодов двухфакторной аутентификации (2FA) для связанных аккаунтов.

## Технологический стек

* **Язык программирования:** Python 3.11
* **Telegram Bot API Framework:** Aiogram 3.28 (асинхронная обработка событий, роутеры, фильтры, конечные автоматы FSM)
* **СУБД и ORM:** PostgreSQL (в продакшене) / SQLite (для тестов), SQLAlchemy 2.0 (асинхронный драйвер asyncpg / aiosqlite, декларативный маппинг Mapped)
* **Кэширование и очереди:** Redis (управление лимитами экспорта, хранение состояний FSM в продакшене, очередь фоновых задач)
* **Асинхронные HTTP-запросы:** Aiohttp 3.9 (клиент-серверная архитектура для работы с внешним API и приема платежных вебхуков)
* **Валидация конфигурации:** Pydantic v2 & Pydantic-Settings (строгий парсинг переменных окружения из файла .env)

## Архитектура и ключевые технические решения

### 1. Точность финансовых расчетов
Все финансовые поля в базе данных (баланс пользователя, сумма покупки, сумма транзакции) используют тип `Numeric(10, 2)` на уровне СУБД и маппятся на тип `decimal.Decimal` в Python. Это полностью исключает накопление погрешностей округления, характерных для типов с плавающей точкой (float / IEEE 754), обеспечивая бухгалтерскую точность транзакций.

### 2. Защита исторических данных (RESTRICT)
Внешние ключи (ForeignKey) таблиц истории транзакций и покупок ссылаются на таблицу пользователей с параметром `ondelete="RESTRICT"`. Это предотвращает случайное каскадное удаление исторических финансовых записей (audit trails) при удалении учетной записи пользователя администратором или системными скриптами. Удаление пользователя заблокировано на уровне СУБД до тех пор, пока существуют связанные с ним финансовые операции.

### 3. Двухфазная верификация привязки аккаунта
Для предотвращения исчерпания пула соединений с базой данных (Connection Pool Starvation) во время выполнения медленных внешних сетевых запросов к API маркетплейса (таймаут до 10 секунд), в хендлере привязки реализован паттерн двухфазной верификации:
* **Фаза 1:** Быстрое открытие транзакции базы данных для проверки баланса и истории покупок. Если условия нарушены (баланс > 1.0 USD или есть покупки), операция отклоняется. Блокировка строки СУБД (`with_for_update`) немедленно снимается, сессия закрывается.
* **Фаза 2:** Выполнение асинхронного HTTP-запроса к API сайта для проверки привязочного хэша вне контекста транзакции БД. Соединение СУБД остается свободным.
* **Фаза 3:** В случае успеха API открывается новая короткая транзакция, которая повторно проверяет состояние пользователя (для исключения race condition при пополнении баланса во время Фазы 2) и фиксирует имя учетной записи.

### 4. Атомарность платежного вебхука
Процесс начисления средств и запись лога транзакции при поступлении платежа от мерчанта объединены в единую атомарную функцию `process_user_deposit` на уровне базы данных. Использование строки `with_for_update` гарантирует защиту от параллельных инкрементов баланса (Race Conditions / Lost Updates), а фиксация изменений внутри одной транзакции СУБД обеспечивает выполнение принципа All-or-Nothing (исключено начисление баланса без создания соответствующей транзакции).

### 5. Защита базы данных от перегрузок (Rate Limiting)
В хендлере экспорта покупок логика Redis-лимитов (максимум 2 запроса в 24 часа) перенесена на самый верх обработчика, до любых обращений к реляционной базе данных. Это защищает PostgreSQL от DoS-атак через спам тяжелыми SQL-запросами на чтение истории, так как Redis отсекает неавторизованные попытки в оперативной памяти.

### 6. Экранирование выводимых данных (Telegram HTML Injection)
Поскольку Telegram-клиент отображает форматированный текст в режиме HTML, все строки, поступающие из внешних API или СУБД (логины внешних сайтов, названия товаров, адреса криптокошельков, 2FA-коды, платежные валюты и методы), принудительно экранируются через встроенный метод `html.escape()`. Это исключает поломку интерфейса из-за некорректных тегов и нейтрализует риски текстового спуфинга и фишинга.

## Структура проекта

```
cypher_bot/
│

├── .env.example               # Шаблон файла конфигурации окружения
├── requirements.txt           # Список зависимостей проекта с фиксацией версий
├── Dockerfile                 # Инструкция сборки Docker-образа для бота и воркера
├── docker-compose.yml         # Конфигурация для запуска Postgres, Redis, Bot и Worker в контейнерах

│
└── bot/
    ├── __init__.py
    ├── config.py              # Парсинг .env на основе pydantic_settings, валидация настроек
    ├── main.py                # Точка входа. Запуск бота, поллинга и встроенного вебхук-сервера aiohttp
    ├── worker.py              # Изолированный фоновый воркер, обрабатывающий очередь экспорта отчетов
    │
    ├── api/
    │   ├── __init__.py
    │   └── client.py          # Асинхронный клиент для взаимодействия с внешним API маркетплейса
    │
    ├── database/
    │   ├── __init__.py
    │   ├── connection.py      # Настройка async_engine, сессий и инициализации СУБД (с поддержкой WAL для SQLite)
    │   ├── models.py          # Декларативные SQLAlchemy модели данных (User, Purchase, Transaction)
    │   └── requests.py        # Атомарные транзакционные операции и SQL-запросы к СУБД
    │
    ├── handlers/
    │   ├── __init__.py
    │   ├── start.py           # Обработка команды /start, переключение языков, навигация по вкладкам
    │   ├── profile.py         # Личный кабинет, привязка аккаунтов (двухфазная), история, экспорт отчетов
    │   ├── balance.py         # Выбор криптовалюты, генерация крипто-инвойсов, валидация USDT-комиссий
    │   └── getcode.py         # Запрос 2FA-кодов для привязанных учетных записей внешнего сайта
    │
    ├── keyboards/
    │   ├── __init__.py
    │   └── inline.py          # Динамические инлайн-клавиатуры с поддержкой языковых локалей
    │
    ├── locales/
    │   ├── __init__.py
    │   └── translation.py     # Локализационные словари (RU/EN) и функции перевода текстов
    │
    └── middlewares/
        ├── __init__.py
        └── i18n.py            # Middleware для автоматического определения локали и инъекции сессий пользователей
```

## Назначение основных модулей

### bot/config.py
Отвечает за централизованную конфигурацию. Класс `Settings` наследуется от `BaseSettings` библиотеки Pydantic, гарантируя строгую типизацию. В случае отсутствия критически важных параметров в `.env` (например, секретов или паролей СУБД) приложение вызовет исключение при старте, не допуская работу в небезопасном режиме.

### bot/database/requests.py
Содержит низкоуровневый интерфейс взаимодействия с СУБД. Все обращения к таблицам строго транзакционны. Функция `process_user_deposit` осуществляет атомарное пополнение баланса с блокировкой строки пользователя, гарантируя неизменность логов транзакций.

### bot/api/client.py
Реализует паттерн обертки над HTTP-клиентом. При работе в тестовом режиме (`DEBUG=True`) автоматически задействует эмуляцию ответа бэкенда маркетплейса для возможности изолированной отладки логики бота.

### bot/middlewares/i18n.py
Анализирует каждый входящий апдейт Telegram. Извлекает идентификатор пользователя, находит или создает запись в базе данных, определяет языковую локаль и внедряет в контекст обработчиков готовую функцию перевода `_()` и экземпляр модели пользователя СУБД.

### bot/worker.py
Изолированная фоновая задача, запускаемая параллельно с основным процессом бота. Выполняет операцию `brpop` из списка `cypher_export_queue` в Redis, выгружает историю покупок пользователя за последние 3 месяца из базы данных, собирает текстовый файл отчета и отправляет его пользователю. Ограничена конкурентной блокировкой `export_lock:{tg_id}` для исключения параллельного запуска экспорта одним и тем же пользователем.

## Конфигурационные переменные (.env)

* `BOT_TOKEN`: Уникальный токен Telegram бота, выданный BotFather.
* `DATABASE_URL`: Строка подключения к PostgreSQL (например, `postgresql+asyncpg://user:pass@host:port/dbname`).
* `REDIS_URL`: Строка подключения к Redis (например, `redis://host:port/0`).
* `DEBUG`: Булевый флаг (`true` или `false`). Активирует SQLite WAL, симуляцию внешних API и подробное логирование.
* `EXTERNAL_API_URL`: Адрес API внешнего сайта маркетплейса.
* `EXTERNAL_API_TOKEN`: Секретный ключ авторизации для работы с внешним API.
* `WEBHOOK_PORT`: Порт, на котором aiohttp-сервер ожидает платежные вебхуки (по умолчанию `8080`).
* `WEBHOOK_SECRET`: Токен заголовка вебхука для верификации подлинности запроса мерчанта.
* `SUPPORT_USERNAME`: Имя пользователя Telegram учетной записи поддержки для перенаправления блокировок привязки.

## Быстрый запуск на Windows (Локальная отладка)

Для максимального удобства локальной разработки в корень проекта добавлен интерактивный диспетчер запуска **`run_cypher_bot.bat`**.

### Возможности лаунчера:
1. **Автоматическое обнаружение окружения**: Скрипт сканирует рабочую директорию на наличие виртуальных сред Python (`.venv`, `venv` или `env`) и автоматически активирует подходящую среду. Если окружение не найдено, запуск происходит через системный интерпретатор Python.
2. **Параллельный запуск процессов (Опция 1 - Рекомендуемая)**:
   - Основной процесс Telegram-бота (`bot/main.py`) и платежный Webhook-сервер на порту `8080` запускаются в текущем окне консоли.
   - Фоновый воркер задач (`bot/worker.py`) автоматически запускается в **новом независимом терминале** с отдельным цветовым выделением (желтый цвет). Это позволяет отслеживать логи генерации файлов отчетов и работу блокировок Redis в реальном времени, не смешивая их с логами сообщений бота.
3. **Управление зависимостями**: Позволяет одной командой обновить менеджер пакетов `pip` и накатить все необходимые библиотеки из `requirements.txt` прямо в активное виртуальное окружение.

### Как запустить:
1. Убедитесь, что настроили файл `.env` (токен бота, доступы к БД и Redis).
2. Запустите двойным кликом файл [run_cypher_bot.bat](file:///d:/Проекты/cypher_bot/run_cypher_bot.bat) в корне проекта.
3. Выберите нужный пункт меню (например, `1` для комплексного запуска бота и воркера) и нажмите Enter.
