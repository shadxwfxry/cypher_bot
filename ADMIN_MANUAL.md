# Руководство администратора Cypher.Bot

Это руководство предназначено для администраторов, контент-менеджеров и операторов техподдержки бота **Cypher.Bot**.

---

## 1. Авторизация и доступ к панели

Для того чтобы получить доступ к панели администрирования, ваш цифровой идентификатор Telegram (User ID) должен быть внесен в переменную `ADMIN_IDS` в файле конфигурации `.env`.

**Пример строки настроек в `.env`:**
```ini
ADMIN_IDS=123456789,987654321
```

Если ваш ID добавлен в этот список, отправьте боту команду `/admin` в чат. Бот пришлет интерактивное меню администрирования. Если вы не являетесь администратором, бот просто проигнорирует данную команду благодаря встроенным фильтрам защиты роутеров.

---

## 2. Разделы панели администрирования

### 📊 Статистика (Statistics)
Позволяет быстро оценить активность аудитории и финансовые показатели за выбранный период (1 день, 7 дней, 30 дней или 90 дней):
*   **New Users / Новые пользователи:** Количество новых пользователей, запустивших бота впервые за этот период.
*   **Purchases / Покупки:** Количество и общая сумма покупок, совершенных пользователями прямо в каталоге бота (в долларах USD).
*   **Deposits / Пополнения:** Количество успешных транзакций пополнения баланса и их суммарный объем (в долларах USD).

### 🔍 Поиск и управление пользователями (User Management)
В данном разделе вы можете найти любого пользователя, зарегистрированного в боте:
1.  Нажмите кнопку **Поиск юзера**.
2.  Отправьте боту никнейм пользователя без `@` (например, `john_doe`) или его численный Telegram ID (например, `123456789`).
3.  Бот выведет полную карточку пользователя:
    *   Связанный логин на основном сайте (site login).
    *   Реферальный параметр запуска (откуда перешел пользователь).
    *   Текущий баланс и статус блокировки.
    *   Общая история покупок и детальные сведения о последней операции.

**Доступные действия в карточке пользователя:**
*   **Изменить баланс (Adjust Balance):** отправьте число, на которое нужно изменить баланс. 
    *   Пример: `25.50` начислит пользователю 25 долларов 50 центов.
    *   Пример: `-10` спишет с баланса пользователя 10 долларов. Баланс не может стать меньше нуля.
*   **Забанить / Разбанить (Ban / Unban):** Мгновенно переключает статус бана. Забаненный пользователь при попытке нажать любую кнопку или отправить команду получит всплывающее уведомление о блокировке.

### ⚙️ Технические работы (Maintenance Mode)
Режим блокировки бота для проведения технических работ на стороне сервера или обновления базы данных.
*   **Включить тех. работы:** Все обычные пользователи при попытке общения с ботом будут получать заглушку: *«Бот временно отключен на технические работы...»*.
*   **Выключить тех. работы:** Снимает ограничение. Бот возвращается в штатный режим работы.
*   *Примечание:* Администраторы из списка `ADMIN_IDS` не подвергаются этой блокировке и могут тестировать функционал бота в реальном времени даже при активном режиме тех. работ.

### 📝 Настройка текстов и кнопок (Edit Settings)
Раздел позволяет на лету переопределять тексты и интерфейс бота без перезапуска кода:
*   **Start Text (RU) / (EN):** Приветственное сообщение, которое видит пользователь при вводе `/start`. Поддерживается HTML-разметка Telegram (`<b>жирный</b>`, `<i>курсив</i>`, `<code>код</code>`).
*   **Button Accounts / Documents (RU) / (EN):** Переопределение надписи на главных кнопках меню («Аккаунты» и «Документы»).

### 📢 Массовая рассылка (Broadcast)
Раздел для отправки новостных объявлений всем пользователям, которые подписаны на рассылку (`is_subscribed=True`):
1.  Нажмите кнопку **Рассылка**.
2.  Отправьте сообщение, которое хотите разослать. Сообщение может содержать форматированный текст (HTML), ссылки и прикрепленную картинку.
3.  Подтвердите отправку кнопкой **Да, запустить**.
4.  Бот начнет фоновую отправку сообщений. Для обхода лимитов Telegram (Flood Control) отправка выполняется асинхронно с задержкой в `0.05 сек` (50 миллисекунд) после каждого получателя.
5.  По окончании рассылки бот пришлет вам отчет с количеством успешно доставленных сообщений и числом пользователей, заблокировавших бота.

## 3. Особенности работы при покупке через API

Для предотвращения рассинхронизации данных между ботом и сайтом маркетплейса при проведении покупок действуют следующие правила:
1.  **Требование привязки (Linkage)**: Совершать покупки в каталоге могут только те пользователи, которые выполнили привязку своего аккаунта к сайту (в меню «Профиль»). Непривязанным пользователям бот выдаст предупреждение с предложением привязать аккаунт.
2.  **Списание через API сайта**: Когда пользователь нажимает «Купить», бот отправляет запрос к сайту. Списание баланса происходит непосредственно на сервере маркетплейса.
3.  **Логирование статистики**: На основе возвращенного ответа бот уменьшает локальный баланс пользователя и заносит записи в локальные таблицы покупок и транзакций, чтобы в разделе **Статистика** всегда отображались актуальные финансовые графики и отчеты.
4.  **Сбои Tor-соединения**: Из-за повышенной латентности Tor-сети покупка может занять до 20-30 секунд. Если сайт вернул ошибку, бот отменит покупку и сообщит об этом пользователю, не изменяя его локальный баланс.
